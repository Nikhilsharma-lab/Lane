# Shadcn Preset Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Lane's custom CSS variable design system with the shadcn preset `b3bZBz3KVM` across the full app.

**Architecture:** Run `shadcn init` on a new branch to install the preset's components and tokens. Add a CSS compatibility shim mapping Lane's old variable names to the new shadcn tokens — this keeps all existing pages rendering immediately without touching every component. Then migrate each page/component group to use proper shadcn Tailwind classes, deleting shim entries as they become unused.

**Tech Stack:** shadcn/ui, Tailwind CSS, Next.js App Router, TypeScript

---

## Variable Mapping Reference

Use this throughout every migration task:

| Lane custom var | Tailwind class (after migration) |
|---|---|
| `var(--text-primary)` / `text-[var(--text-primary)]` | `text-foreground` |
| `var(--text-secondary)` / `text-[var(--text-secondary)]` | `text-muted-foreground` |
| `var(--text-tertiary)` / `text-[var(--text-tertiary)]` | `text-muted-foreground/60` |
| `var(--bg-base)` | `bg-background` |
| `var(--bg-surface)` | `bg-card` |
| `var(--bg-subtle)` / `var(--bg-section)` | `bg-muted` |
| `var(--bg-hover)` / `var(--bg-active)` | `bg-accent` |
| `var(--border)` | `border` (Tailwind default) |
| `var(--border-strong)` | `border-border/80` |
| `var(--accent)` — Lane's green action color | `text-primary` / `bg-primary` |
| `style={{ color: "var(--accent)" }}` | replace with `className="text-primary"` |
| `style={{ background: "var(--accent)" }}` | replace with `className="bg-primary"` |

**Important:** After init, shadcn's `--accent` token is a hover-state background, NOT Lane's green. Every `var(--accent)` usage must be explicitly migrated to `var(--primary)` or Tailwind's `text-primary`/`bg-primary`.

- [ ] **Step 2: Replace the entire old `:root {}` block**

  --bg-base:     var(--background);
